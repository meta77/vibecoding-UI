# VibeCoding UI Dictionary 実装計画

## 目的
AIと対話しながらコードを生成するスタイル（VibeCoding）における、「UIパーツの名称と説明」および「AIへの効果的な指示方法」をまとめたシングルページのWebアプリケーションを構築する。

## 技術スタック
- **フレームワーク**: Vite + Vue
- **スタイリング**: Tailwind CSS
- **デザイン**: モダンなダークカラー基調
- **言語**: TypeScript

## ユーザーレビューが必要な項目
> [!NOTE]
> Vercelでの公開を目標とするため、Vite + Vue 3を採用し、Vercelデプロイ用に設定を行います。

## 提案する変更内容

### 1. プロジェクトの初期化
- Viteを使用して、Vue + TypeScriptのプロジェクトをルートディレクトリにセットアップする。
- Tailwind CSSをインストールし、設定を行う。

### 2. コンポーネント設計 (UI)
ダークテーマを基調とし、以下の2つのタブを持つ1ページのUIを構築する。
- **タブ1: 一般的なWeb UIパーツ**
  - ボタン、カード、モーダル、フォームなどの基本パーツの名称、役割、よくある実装パターンの解説。
  - 各UIパーツに対する「AIへの効果的な指示（プロンプト）のベストプラクティス」を併記する。
- **タブ2: Tailwind CSS UIパーツ**
  - Tailwind CSSを用いた具体的なユースケースや、ユーティリティクラスの組み合わせによるコンポーネント実装の解説。
  - Tailwindで特定のデザインを再現する際の、「AIへの効果的なプロンプトの出し方」を併記する。

### 3. デザイン・スタイリング (Tailwind CSS)
- `index.css` にてダークテーマのカラースキーム（背景: `#111827`など、テキスト: `#f9fafb`など）を定義。
- タブ切り替えのアニメーションや、ホバーエフェクトなどのマイクロインタラクションを追加し、洗練されたモダンな印象を持たせる。

## 検証計画

### 手動検証
1. 開発サーバー (`npm run dev`) を立ち上げ、ブラウザでUIが正常に表示されるか確認する。
2. ダークテーマが正しく適用されているか、高いコントラストと視認性が確保されているか確認する。
3. 3つのタブがスムーズに切り替わり、それぞれのコンテンツが正しく表示されるか確認する。
### 配信検証
- Vercelにデプロイできる構成になっているかを検証する。
