# VibeCoding UI Dictionary 実装計画 (ジャンル別UIパーツ拡張)

## 目的
「General Components」タブ内をジャンルごとに整理し、今回は「Input & Forms」ジャンルに5つのUIパーツを追加する。
「Action & Trigger」と同様に、プロ仕様の解説とAIプロンプト、デモコンポーネントを実装する。

## ユーザーレビューが必要な項目（要件の確認）
> [!IMPORTANT]
> 「ジャンルを一つ選択し、その各ジャンルごとに5つずつUIパーツを説明する構成」について、以下の点を確認させてください。
> 
> 1. **UIの構成方法**: 「General Components」タブを開いた際、左側にジャンル一覧（サイドバー）、右側にそのジャンルに属する5つのUIパーツが並ぶような構成（または上部にサブタブ）でよろしいでしょうか？
> 2. **実装の範囲**: 今回のステップでは、**全てのジャンル（5ジャンル × 各5パーツ ＝ 25パーツ）** を一気に実装しますか？それとも、**まずは代表的な1ジャンル（例：Input & Formsのみ、5パーツ）** を実装し、デザインや構成のレビューを行いますか？
> 3. **AIプロンプト例の扱い**: 基本的には、これまで通り「各UIパーツの解説＋AIプロンプト」のセットを維持する方針でよろしいでしょうか？

## 提案する変更内容

### 1. データ構造の変更 (`src/App.vue`)
- 現在フラットな配列になっている `generalParts` を、ジャンルごとのオブジェクトを格納した構造に変更する。
- データの例：
  ```ts
  const genres = [
    {
      id: "input",
      name: "Input & Forms",
      description: "ユーザーから情報を受け取るためのパーツ。",
      parts: [
        { name: "Text Input", description: "...", prompt: "...", demoComponent: "textInput" },
        // ... (計5パーツ)
      ]
    },
    // ... 他のジャンル
  ]
  ```

### 2. UIレイアウトのアップデート (`src/App.vue`)
- `activeTab === 'general'` の表示領域を、**ジャンル選択エリア（サイドバー）** と **パーツ表示エリア（メインエリア）** の2カラム構成にする。
- **パーツ表示エリアの2列グリッド化**: Web（デスクトップ）閲覧時にパーツカードが2列（`grid-cols-2`）で並び、モバイルでは1列（`grid-cols-1`）になるようにレスポンシブなグリッドレイアウトを採用する。
- **2列表示に伴うデザイン最適化**:
  1. 幅が半分になるため、カードごとのパディングやフォントサイズをややコンパクトにし、間延びしないように調整する。
  2. デモエリア（イメージ）の比率（アスペクト比）を維持しつつ、圧迫感のない高さを確保。
  3. テキスト構造（名称→イメージ→説明→プロンプト）の順序は維持したまま、カード全体を囲む控えめな境界線または背景色を設け、1枚の独立した情報ブロックとしてのまとまりを強化する。

### 3. スタイルの維持
- 現在の「Soft Nordic Dark」のトンマナ（`neutral-900` 背景、パールホワイトのアクセント、ソフトグローなど）を維持したまま、サブナビゲーションを美しく組み込む。
### 4. Input & Forms コンテンツの追加
- 以下の5つのパーツを実装予定：
  1. **Text Input (テキスト入力)**: 基本的なテキスト情報の入力。
  2. **Checkbox (チェックボックス)**: 複数選択。
  3. **Radio Button (ラジオボタン)**: 単一選択。
  4. **Toggle Switch (トグルスイッチ)**: 設定のON/OFF。
  5. **Select Menu (セレクトメニュー)**: 選択肢のリストからの選択。
- 各パーツに「Nordic Dark」なデモと、AIへの効果的なプロンプトを追加する。
## 検証計画
- `npm run dev` にて、ジャンル切り替えがスムーズに行えるか検証する。
- モバイルビューで、サイドバー（またはサブタブ）が使いやすくレスポンシブに折りたたまれるか検証する。
