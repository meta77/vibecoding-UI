# VibeCoding UI Dictionary 実装計画 (ジャンル別UIパーツ拡張)

## 目的
「General Components」タブ内をジャンルごとに整理し、ユーザーがジャンルを選択して、そのジャンルに属する5つのUIパーツを閲覧できる構成に変更する。

## ユーザーレビューが必要な項目（要件の確認）
> [!IMPORTANT]
> 「ジャンルを一つ選択し、その各ジャンルごとに5つずつUIパーツを説明する構成」について、以下の点を確認させてください。
> 
> 1. **UIの構成方法**: 「General Components」タブを開いた際、左側にジャンル一覧（サイドバー）、右側にそのジャンルに属する5つのUIパーツが並ぶような構成（または上部にサブタブ）でよろしいでしょうか？
> 2. **実装の範囲**: 今回のステップでは、**全てのジャンル（5ジャンル × 各5パーツ ＝ 25パーツ）** を一気に実装しますか？それとも、**まずは代表的な1ジャンル（例：Input & Formsのみ、5パーツ）** を実装し、デザインや構成のレビューを行いますか？
> 3. **AIプロンプト例の扱い**: 基本的には、これまで通り「各UIパーツの解説＋AIプロンプト」のセットを維持する方針でよろしいでしょうか？

## 提案する変更内容

### 1. データ構造の変更 (`src/App.vue`)
- 現在フラットな配列になっている `generalParts` を、ジャンルごとのオブジェクトを格納した構造に変更する。
- データの例：
  ```ts
  const genres = [
    {
      id: "input",
      name: "Input & Forms",
      description: "ユーザーから情報を受け取るためのパーツ。",
      parts: [
        { name: "Text Input", description: "...", prompt: "...", demoComponent: "textInput" },
        // ... (計5パーツ)
      ]
    },
    // ... 他のジャンル
  ]
  ```

### 2. UIレイアウトのアップデート (`src/App.vue`)
- `activeTab === 'general'` の表示領域を、**ジャンル選択エリア（サブナビゲーション）** と **パーツ表示エリア（メインエリア）** の2カラム構成（またはサブタブ切り替え）にアップグレードする。
- 選択されたジャンルに応じて、配下の5つのUIパーツがグリッド（またはリスト）で表示されるようにする。

### 3. スタイルの維持
- 現在の「Soft Nordic Dark」のトンマナ（`neutral-900` 背景、パールホワイトのアクセント、ソフトグローなど）を維持したまま、サブナビゲーションを美しく組み込む。

## 検証計画
- `npm run dev` にて、ジャンル切り替えがスムーズに行えるか検証する。
- モバイルビューで、サイドバー（またはサブタブ）が使いやすくレスポンシブに折りたたまれるか検証する。
