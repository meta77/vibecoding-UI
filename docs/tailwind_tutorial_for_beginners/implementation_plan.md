# Tailwind UI 構築チュートリアル 実装計画（3部構成・導入編の追加）

## 目的
既存のチュートリアルを大改修し、HTML/CSSの基礎知識がある開発者向けに「Tailwindの導入からモダンUIの実装まで」を網羅する3部構成の総合ガイドへと進化させる。

## 新しいチュートリアル構成案（全3章）

### 第1章：Tailwindの始め方（セットアップ編）
HTML/CSSの基礎を理解している方向けに、モダンなフロントエンド開発環境（Vite + Tailwind v4）を迅速に構築するステップを追加します。
- **Step 1: Viteプロジェクトの作成とインストール**
  - 内容: `npm create vite@latest` と Tailwind v4 モジュールのインストールコマンドの解説。
- **Step 2: Tailwind v4 の有効化**
  - 内容: `vite.config.ts` へのプラグイン追加と、`style.css` への `@import "tailwindcss";` の記述によるシンプルな設定手法の解説。

### 第2章：Tailwindの哲学、コツ（理論編）
※既存のステップを整理し、本質的な設計思想を学びます。
- **Step 3:** ユーティリティファーストの哲学（Tailwindの存在意義）
- **Step 4:** 空間の支配 (PaddingとMargin)
- **Step 5:** 要素のグループ化と空間 (Gapの魔法・近接の法則)
- **Step 6:** タイポグラフィと色彩心理（Nordic Darkの色使い）
- **Step 7:** 質感の構築 (角丸・ボーダー・影)
- **Step 8:** 心地よいインタラクション（動きと変化）

### 第3章：よくあるUIパーツの具体例（Nordic Dark実践編）
※既存の実践ステップを「具体例集」として独立させます。
- **Step 9:** Primary Button の構築
- **Step 10:** Status Badge の構築
- **Step 11:** Container Card の構築

## UI実装方針 (`App.vue`)
- 現在のフラットな配列 `tutorialSteps` を、**「章（Chapter）」の概念を持たせたデータ構造に変更**（例: `{ chapterTitle: "第1章...", steps: [...] }`）。
- 画面上で章ごとに大きな見出しエリア（セクション区切り）が表示されるよう、UIのテンプレートレンダリングを調整します。
- Step 1〜2のデモコンポーネントとして、「ターミナル」や「コードエディタ」を模したグラフィカルなUI（Nordic Dark風）を新規作成します。

## ユーザーレビューが必要な項目
> [!IMPORTANT]
> - 「Tailwindの始め方 → 哲学・コツ → UI具体例」という3部構成のカリキュラムと、Vite + Tailwind v4 を前提とした導入手法としております。こちらの全体構成でよろしいでしょうか？
